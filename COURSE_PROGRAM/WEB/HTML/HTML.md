**Зачем нужны теги
HTML теги - это основа, каркас страницы. Как дом состоит из кирпича так и сайт состоит из тегов.
Но так же с помощью некоторых тегов можно сделать то, что невозможно без них. Например добавить изображение на страницу, кнопку, ссылку и т.д.
В html'e есть два вида тегов: технические и контентные.

Технические нужны для настроек. Настройки кодировки, подключение файлов стилей, подключение скриптов, указание ключевых слов и многое другое, что не является контентом страницы и не выводиться на нее.

Контентные - для управления содержимом страницы. А сюда входят все остальные теги: добавление изображений, указания ссылки, задания заголовков страницы, выделение текста и т.д.

Значение атрибута может записыватся и без ковычек но если у вас больше одного значения то браузер не сможет нормально это воспринять.

** Ссылки
Ссылки создаются с помощью тега <a></a>

У тега есть обязательный атрибут href="", значение которого может быть адрес или файл, который будет открыватся после нажатие на ссылку.

<a href="http://yandex.ru">Открыть сайт</a>

По-умолчанию в браузере текст ссылки будет отображатся синего цвета и иметь подчеркивание. Так же открыватся ссылка будет в этом же окне.

У тега есть дополнительные атрибуты
target="" - в каком окне будет открыватся ссылка. Если мы хотим, чтобы открытие было в новом окне то значение атрибута должно содержать текст "_blank".

Ссылкой может быть не только текст но и картинка а так же любые другие html элементы. Для этого надо картинку обернуть в тег <a>

<a href="http://yandex.ru">
	<img src="image.jpg">
</a>

Якорыне ссылки

Ссылка может быть не только на сторонний ресурс или файл но и на внуренне содержимое документа.

Чтобы сделать ссылку на какой-то блок на странице первым делом этому блоку надо задать атрибут ID с каким-нибудь значением

<p id="text-with-image">
	Тут какой-то текст с картинками.
</p>

а зачем создать ссылку на этот блок

<a href="#text-with-image">Ссылка на текст с картинками</a>

Если ссылка идет на ID'шник то она должна начинаеться со знака "#" (hash, sharp или octotorb :) ).

Ссылка может быть ниже или выше это роли не играет. При нажатии на ссылку браузер промотает до соответствующий области где находится блок. Так же в адресной строки браузера добавиться текст "#text-with-image".

Теперь если вы скопируете URL и откроете его в новом окне то браузер автоматически промотает до соответствующего блока.

Основной тег для ввода данных называется input. Он ординарный.

type="password" - элемент для ввода текста но вместо текста будут маркеры.

type="radio" - элемент где можно выбрать одни из вариантов. Используется как правило несколько так как в одном элементе нет смысла (Например, Выберете пол: м, ж. Или в тесте выберете один из вариантов) 
type="file" - элемент для передачи файла
type="submit" - элемент кнопка.

** Семантические теги
Раньше для оформления страницы использовали такие теги как: h1-h6, p, b, i, hr и т.д. Сейчас все делается через CSS.

Например надо сделать текст "Привет мир" красным, жирным и по середине.
Как это было раньше:
<center>
	<font color="red">
		<b>Привет мир</b>	
	</font>	
</center>

Как это стало в CSS
<div style="color: red; text-align: center; font-weight: bold">Привет мир</div>

Итог: сокращение кода, избавление от ненужной вложенности тегов.

И теперь для оформления страницы эти теги не нужны. Но какая роль у них сейчас и что значит семантика ?

Когда вы выложите свой сайт на хостинг вам надо будет его проиндексировать в поисковых системах, чтобы его можно было найти.
Поисковые системы с помощью своих скриптов(роботов) сканируют сайт. И есть сайт будет состоять целиком из div тегов то не будет понятно, что есть что. Где заголовок, где параграф, где шапка сайта где подвал и т.д.
Поэтому старые теги переквалефициорвались в семантические и добавились новые.
Из старых:
Заголовок - h1
Параграф - p
Из новых:
header - шапка сайта
footer - подвал сайта

В процессе верстки вы можете сразу создать тег header если вы знаете, что это будет шапка и в будушем не поменяется но некоторые это делают перед сдачей проекта.

Было
<div class="block-header">
	<div class="links">
		<div class="link-item">Главная</div>
		<div class="link-item">Новости</div>
		<div class="link-item">Контакты</div>
	</div>
</div>

Стало
<header class="block-header">
	<nav class="links">
		<div class="link-item">Главная</div>
		<div class="link-item">Новости</div>
		<div class="link-item">Контакты</div>
	</nav>
</header>

Замена тегов на отоброжение никак не влияет.
